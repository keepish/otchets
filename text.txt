Используя draw.io / Visio / Figma создать wireframe для разрабатываемой системы. 
Количество экранов — не менее 10. На wireframe должны быть показаны переходы 
между экранами и обозначены основные элементы управления и возможности каждой 
категории пользователей.

Вопросы для уточнения информации о разрабатываемом продукте:
Пользователи и доступ
1.	Какие категории пользователей будут работать с системой (администраторы, сотрудники центра, безработные)?
2.	Какие действия будут доступны для каждой конкретной роли?
3.	Каким образом планируется организовать регистрацию и аутентификацию пользователей?
4.	Нужна ли интеграция с существующими государственными порталами (например, «Госуслуги»)?
Архитектура и работа ПО
5.	Планируется ли использовать веб-приложение, настольное ПО или мобильное приложение?
6.	Есть ли у центра занятости уже используемое оборудование (серверы, ПК), которое нужно учитывать?
7.	Каким образом пользователи будут получать доступ к системе (локально или через интернет)?
8.	Предусматривается ли работа системы в оффлайн-режиме?
Данные и хранение
9.	Нужно ли хранить историю трудоустройства каждого человека?
10.	Какие данные должны быть доступны только сотрудникам, а какие — пользователям?
11.	Нужно ли хранить документы (сканы дипломов, трудовых книжек)? 
12.	Необходима ли фильтрация списка безработных? По каким параметрам? 
13.	В каком виде должны отображаться безработные (многострочный список со всеми данными, в виде карточек)?
Отчёты и документы
14.	Какие типы отчётов должны формироваться системой (по возрасту, полу, образованию и т. д.)?
15.	В каких форматах требуется экспорт отчётов?
16.	Какие входящие документы должны загружаться в систему?
17.	Нужно ли автоматическое формирование писем/уведомлений для пользователей?
Протоколы и интеграции
18.	Нужно ли использовать электронную подпись?
19.	Какой уровень безопасности данных необходим (шифрование, резервное копирование)?
20.	Необходима ли в приложении реализация светлой/темной темы? 
Интервьюирование
1.	Администраторы, сотрудники центра занятости, безработные, работодатели.
2.	Администратор: управление пользователями, настройка системы, резервное копирование. 
Сотрудник центра: регистрация и ведение учёта безработных, формирование отчётов, работа с документами.
Безработный: регистрация в системе, загрузка документов, просмотр вакансий, подача заявок.
Работодатель: размещение вакансий, просмотр откликов, приглашение соискателей.
3.	Регистрация через портал «Госуслуги», с возможностью локальной регистрации для сотрудников. Двухфакторная аутентификация для сотрудников и администраторов.
4.	Да, с «Госуслугами» для подтверждения личности и с налоговой службой для проверки факта трудоустройства.
5.	Веб-приложение с адаптивным интерфейсом.
6.	Да, есть сервер и локальная сеть, которые можно использовать для установки системы. Рабочие места сотрудников оснащены ПК.
7.	Через интернет, с использованием защищённого соединения HTTPS.
8.	Для сотрудников – частично, с возможностью работы в локальной сети. Для безработных и работодателей доступ только онлайн.
9.	Да, обязательно (дата приёма и увольнения, предприятие, должность).
10.	Сотрудникам: паспортные данные, семейное положение, история трудоустройства.
Пользователям: контактные данные работодателей.
11.	Да, для подтверждения квалификации и образования.
12.	По возрасту, полу, образованию, профессии, числу трудоустроенных за период, динамике обращений.
13.	PDF – для отправки во внешние органы; XLSX – для внутреннего анализа; DOCX – для печати.
14.	Скан паспорта, диплома, свидетельства о рождении детей, трудовой книжки, ИНН, СНИЛС.
15.	Да, уведомления о новых вакансиях, изменении статуса заявки, приглашениях от работодателей.
16.	Да, для заверения документов сотрудников и подтверждения согласия соискателей.
17.	Высокий: шифрование персональных данных, резервное копирование, разграничение прав доступа, аудит действий пользователей.
18.	Да.
19.	Да. По возрасту, полу, образованию, профессии, дате постановки на учёт, статусу трудоустройства.
20.	В виде карточек.
